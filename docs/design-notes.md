# 设计笔记

这里记录几个关键设计选择与已知的近似，方便后续迭代时对照。

## 为什么这样做音色解耦

单一手段往往不足以把说话人信息从内容里剥离，Timbrel 叠加了三种互补机制：

1. **实例归一化瓶颈**：内容编码器末端对每个通道在时间维做实例归一化，抹掉句级（近似说话人级）统计，是 AdaIN-VC 一系的思路。
2. **对抗说话人分类器 + 梯度反转**：在池化后的内容表征上接一个说话人分类器，经梯度反转层（DANN）促使编码器学到说话人无关的内容。
3. **CLUB 互信息上界**：以变分方式估计 `I(content; speaker)` 的上界并最小化，作为对抗之外的显式约束。变分网络自身用似然目标单独训练，其梯度不回流编码器。

三者的权重（`lambda_adv`、`lambda_club`）都放在 `TrainConfig` 里，便于消融。

## 说话人自适应放在条件 LayerNorm

音色只在**解码阶段**通过条件 LayerNorm（AdaSpeech）注入：网络主体保持说话人无关，身份信息集中在少量 scale/bias 参数上。这样做的直接好处是——少样本自适应时只需微调这部分参数（见 `adaptation.freeze_for_adaptation`），参数量小、过拟合风险低。

## 韵律：FiLM + 对抗解耦

韵律编码器把参考句压成一个韵律向量，再映射为 FiLM 的 `gamma/beta` 去调制内容。由于该向量同样经对抗训练做了说话人解耦，推理时可以从一段参考取韵律、从另一段取音色，实现韵律迁移。

## 已知近似与局限

- **英文 g2p**：内置小词典 + 字母规则回退，仅为占位；后续计划接入 CMUdict / 神经 g2p。
- **三声变调**：只实现了连续三声的左向近似，未覆盖「一 / 不」及更复杂的韵律条件变调。
- **拼音切分**：声母/韵母按最长前缀切分，`j/q/x/y` 后的 `ü`（写作 `u`）按普通 `u` 处理，是发音上的近似。
- **f0 估计**：时域自相关，`O(n^2)`，仅适合离线构建韵律目标；长音频建议换 FFT 版自相关或成熟的 pyin/DIO。
- **基频/能量**：在**音素级**预测（FastPitch 风格），比帧级更简，但牺牲了帧内细节。
- **声码器缺失**：当前只到 mel，接 HiFi-GAN 等声码器才能出波形。
